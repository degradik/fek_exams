Системы счисления

Основание системы показывает, сколько цифр используется. В двоичной — 0 и 1; в восьмеричной — 0–7; в шестнадцатеричной — 0–9 и A–F, где A=10, B=11, …, F=15.

Из любой системы в десятичную

  1. Пронумеруйте разряды справа налево, начиная с нуля.
  2. Умножьте каждую цифру на основание в степени её разряда.
  3. Сложите результаты.
101101₂ = 1·2⁵ + 0·2⁴ + 1·2³ + 1·2² + 0·2¹ + 1·2⁰ = 32 + 8 + 4 + 1 = 45₁₀

Из десятичной системы в другую

  1. Делите число на новое основание.
  2. Каждый раз записывайте остаток.
  3. Продолжайте, пока частное не станет нулём.
  4. Прочитайте остатки снизу вверх.
45 / 2 = 22, остаток 1 22 / 2 = 11, остаток 0 11 / 2 = 5, остаток 1 5 / 2 = 2, остаток 1 2 / 2 = 1, остаток 0 1 / 2 = 0, остаток 1 Ответ: 101101₂

Быстрый перевод 2 ↔ 8 и 2 ↔ 16

Для восьмеричной системы разбивайте двоичное число справа на группы по 3 бита, для шестнадцатеричной — по 4 бита.

110 101 ₂ = 65₈ 1101 0110 ₂ = D6₁₆
Всегда делайте обратную проверку переводом результата в десятичную систему.

Задания с IP-адресом и маской

IPv4-адрес содержит 32 бита — четыре числа от 0 до 255. Маска тоже содержит 32 бита: сначала идут единицы — часть сети, затем нули — часть узла.

Адрес сети = IP-адрес AND маска

Самый простой случай: маска /24

/24 означает 24 единицы и 8 битов для узлов. Маска: 255.255.255.0.

IP: 192.168.10.37 Маска: 255.255.255.0 Сеть: 192.168.10.0 Broadcast: 192.168.10.255 Узлы: 192.168.10.1 — 192.168.10.254

Универсальный алгоритм

  1. Если дана запись /n, найдите число битов узла: 32 − n.
  2. Количество адресов в подсети: 2^(32−n).
  3. В обычной задаче доступных узлов: 2^(32−n) − 2 — адрес сети и broadcast не выдаются узлам.
  4. Для изменяющегося октета вычислите размер блока: 256 − значение маски.
  5. Найдите интервал блока, куда попадает октет IP. Начало — адрес сети, конец — broadcast.

Пример с /26

/26 = 255.255.255.192. Размер блока: 256 − 192 = 64. Границы подсетей: 0, 64, 128, 192.

IP: 192.168.1.130 Маска: 255.255.255.192 (/26) 130 попадает в блок 128–191 Сеть: 192.168.1.128 Broadcast: 192.168.1.191 Узлы: 192.168.1.129 — 192.168.1.190 Адресов: 2⁶ = 64; обычных узлов: 64 − 2 = 62

Побитовое AND

Если нужна точная проверка, переведите изменяющийся октет в двоичный вид. В результате AND единица остаётся только там, где единицы есть в обоих числах.

130 = 10000010 192 = 11000000 AND = 10000000 = 128
Не путайте: адрес сети — первый адрес блока, broadcast — последний. Первый и последний обычный узел находятся между ними.

Единицы информации

1 байт = 8 бит 1 Кбайт = 1024 байта 1 Мбайт = 1024 Кбайта 1 Гбайт = 1024 Мбайта

Если алфавит содержит N равновероятных символов, минимальное число бит на символ находят из условия 2ⁱ ≥ N. Объём сообщения: количество символов × бит на символ.

Графы и поиск

DFS — поиск в глубину: идёт по одному пути как можно дальше. BFS — поиск в ширину: обходит вершины слоями.

  • Достижимость можно проверить DFS или BFS.
  • Кратчайший путь по числу рёбер в невзвешенном графе находит BFS.
  • Двоичный поиск работает только в отсортированном массиве и имеет сложность O(log n).

Базы данных

Таблица состоит из полей — столбцов — и записей — строк. Первичный ключ однозначно идентифицирует запись и не должен повторяться.

Запрос выбирает данные по условию. Связь «один ко многим» означает, что одной записи первой таблицы соответствует несколько записей второй.
